# UI/運用の改善提案

現状の「年/月/日/曜日/時刻」の混在が分かりづらい点を解消するための案です。

## 1. 入力モードを分ける
- `実行タイプ` 列を追加（`日付指定` / `繰り返し`）。選択に応じて必須列を決める。
- シート上部に「モード別の必須項目」の短い表を置き、迷わないようにする。

## 2. 日付指定の簡素化
- 年月日を1セルの「日付」列にまとめ、時刻は別列で日付型に限定。部分指定は廃止し「一度きり」にする。
- 繰り返し通知は別モードに切り出し、日付指定と混在させない。

## 3. 繰り返しパターンを明示
- `繰り返しパターン` 列（`毎週` / `毎月` / `毎年` など）を設ける。
- 毎週: 曜日チェックボックス必須。年/月列は置かない（入力させない）。
- 毎月: `日` 列のみ必須。年/月は入れさせない。
- 毎年: `月` + `日` を必須。年は不要。

## 4. 休日・その他オプションの整理
- `土日祝も通知する` チェックボックスを1列に統一し、false なら土日祝を一律スキップと明記。
- 宛先（旧メンション）は `here`/`channel`/ユーザー名またはIDのカンマ区切りに限定と明記。
- ワークスペースは単一前提に固定し、チーム列はなくす（Webhook は環境変数/設定に1つだけ）。

## 5. 見た目と入力規則
- 必須列に色付け、データ検証で日付型・時間型・ドロップダウンを強制。
- 例行（繰り返し）と単発（日付指定）を別シートに分ける運用も有効。

## 6. コード側のシンプル化に繋げる
- モードとパターンが明示されれば、判定ロジックを単純な分岐にでき、年/月混入による取りこぼしを防げる。
